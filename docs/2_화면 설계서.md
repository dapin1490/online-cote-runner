# 화면 설계서 (Wireframe Specification)

## 1. 레이아웃 전략 (Layout Strategy)

- 프레임워크: Bootstrap 5 Grid System.
- 구조:
    - Desktop (md 이상): 2-Column Split View (좌측 9 : 우측 3, 드래그하여 조절 가능).
    - Mobile (sm 이하): 1-Column Stack View (상단 에디터 -> 하단 입출력).
- 테마: 개발자 친화적인 Dark Mode 권장.

## 2. 세부 화면 구성 (Detailed UI Components)

### 2.1 헤더 (Header / Navigation Bar)

상단에 고정(`sticky-top`)된 네비게이션 바입니다.

- 좌측: 로고 및 프로젝트명 (Text: "Online Coding Test Runner").
- 중앙: 언어 선택 드롭다운 (Select Box).
    - 옵션: `C++`, `Python`.
- 우측: 기능 버튼 그룹.
    - `[Share]` 버튼: 클릭 시 현재 상태 URL 복사. (아이콘: 🔗)

### 2.2 메인 컨테이너 - 좌측 패널 (Code Editor Area)

소스 코드를 작성하는 영역입니다.

- 구성: `div#editor-container`
- 컴포넌트: CodeMirror 6 인스턴스.
- 스타일:
    - 높이: `calc(100vh - 56px)` (헤더 높이 제외한 전체 높이).
    - 폰트: Monospace (등폭 글꼴).
    - 줄 번호(Line Numbers) 활성화.

### 2.3 메인 컨테이너 - 우측 패널 (Interaction Area)

테스트 케이스를 관리하고 실행 결과를 확인하는 영역입니다. 상-중-하 3단 구조를 가집니다.

### A. 상단: 테스트 케이스 탭 (Test Case Tabs)

- 컴포넌트: Bootstrap Nav Tabs (`ul.nav.nav-tabs`).
- 구성:
    - `Case 1` | `Case 2` | ... | `[+]` (추가 버튼).
    - 각 탭 활성화 시 `[x]`(삭제) 버튼 노출 (단, 탭이 1개일 때는 삭제 불가).

### B. 중단: 입출력 입력창 (IO Inputs)

선택된 탭에 종속되는 입력 영역입니다.

- Label: "Standard Input (Stdin)"
- Input: `<textarea>` (Placeholder: 입력값을 넣으세요).
- Label: "Expected Output"
- Input: `<textarea>` (Placeholder: 정답 기대값을 넣으세요).

### C. 하단: 실행 제어 및 결과 (Controls & Result Console)

- Action Bar:
    - `[Run All]` 버튼: 너비 100% (`btn-primary`, `w-100`).
- Result Area:
    - 초기 상태: 비어 있음 또는 "실행 버튼을 눌러주세요".
    - 실행 후:
        - 종합 요약: "Total: 3, Pass: 2, Fail: 1" (Card 형태).
        - 개별 결과 (Accordion List):
            - Header: `Case 1: ✅ Pass (0.02s)` / `Case 2: ❌ Fail` / `Case 3: ⚠️ Error`.
            - Body (Fail/Error 클릭 시 펼침):
                - *Fail 시:* `Diff View` (좌: 내 출력 / 우: 정답).
                - *Error 시:* `stderr` 텍스트 (색상: Red).

## 3. 텍스트 목업 (Text Mockup)

이해를 돕기 위해 데스크톱 기준 레이아웃을 시각화했습니다. 아래 목업은 공간 상 문제로 좌우 비율이 동일하지만, 데스크톱 기준 실제 화면은 좌우 비율 9:3입니다. 또한 사용자가 드래그하여 조절할 수 있어야 합니다.

```
+---------------------------------------------------------------+
| [Logo] Online Coding Test Runner [ Select: Python v ] [Share] |  <-- Header
+---------------------------------------------------------------+
| 1  def solve():             | [ Case 1 ] [ Case 2 ] [ + ]     |  <-- Tabs
| 2      a, b = map(int...    |---------------------------------|
| 3      print(a + b)         | Standard Input:                 |
| 4                           | [ 10 20                       ] |
| 5                           |                                 |
| 6                           | Expected Output:                |
| 7                           | [ 30                          ] |
| 8                           |                                 |
| 9                           |---------------------------------|
| .                           | [       RUN ALL (Button)      ] |  <-- Action
| .                           |---------------------------------|
| .                           | Result: Total 2, Pass 1, Fail 1 |  <-- Summary
| .                           | [v] Case 1: ✅ Pass (0.05s)    |
| .                           | [>] Case 2: ❌ Fail (0.04s)    |  <-- Accordion
|                             |     My Output: 40               |
|                             |     Expected:  50               |
+-----------------------------+---------------------------------+
```

## 4. 인터랙션 상태 정의 (Interaction States)

1. 로딩 중 (Running):
    - `[Run All]` 버튼이 `Running...`으로 텍스트 변경 및 비활성화(Disabled).
    - 스피너(Spinner) 아이콘 표시.
2. 공유 (Sharing):
    - `[Share]` 버튼 클릭 시:
        - URL 해시 업데이트.
        - 브라우저 `alert` 또는 `toast` 메시지: "Link copied to clipboard!".
3. 반응형 (Mobile):
    - 화면 너비 768px 미만 시, 우측 패널이 좌측 패널(에디터) 아래로 이동.
    - 에디터 높이는 `50vh`로 조정.
