# 코드 컨벤션 (Code Convention)

## 1. 디렉토리 및 파일 구조 (Directory Structure)

GitHub Pages 배포 환경을 고려하여 루트(Root)에 `index.html`을 배치하고, 정적 자원은 `assets` 폴더에서 관리합니다.

```
/ (Root)
├── index.html          # 메인 페이지 (진입점)
├── README.md           # 프로젝트 설명서
└── assets/
    ├── css/
    │   └── style.css   # 커스텀 스타일 (Bootstrap 오버라이딩)
    ├── js/
    │   ├── app.js      # 메인 로직 (이벤트 핸들러, 초기화)
    │   └── api.js      # API 통신 모듈 (Piston API)
    └── images/         # (필요 시) 로고, 파비콘 등
```

## 2. 명명 규칙 (Naming Conventions)

| **대상** | **스타일** | **예시** | **비고** |
| --- | --- | --- | --- |
| **변수 / 함수** | `camelCase` | `userCode`, `runTestCases()` | 동사+명사 조합 권장 |
| **상수** | `UPPER_SNAKE_CASE` | `MAX_TEST_CASES`, `API_URL` | 변경되지 않는 값 |
| **클래스** | `PascalCase` | `TestCaseManager` | (필요 시 사용) |
| **HTML ID / Class** | `kebab-case` | `code-editor`, `btn-run-all` | Bootstrap 클래스 스타일과 통일 |
| **파일명** | `kebab-case` | `api-client.js`, `style.css` | 소문자 및 하이픈 사용 |

## 3. JavaScript 코딩 스타일 (ES6+ 권장)

### 3.1 변수 선언

- `var` 사용을 금지합니다.
- 값이 변하지 않으면 `const`, 변하면 `let`을 사용합니다.

```jsx
// Bad
var x = 10;

// Good
const MAX_COUNT = 5;
let currentIndex = 0;
```

### 3.2 문자열 처리

- 일반 문자열은 작은따옴표(`'`) 또는 큰따옴표(`"`) 중 하나로 통일하되, 템플릿 리터럴(백틱, ```) 사용을 적극 권장합니다.

```jsx
// Good (Template Literal)
const message = `Test Case ${index + 1}: Pass`;
```

### 3.3 비동기 처리

- `Promise` 체이닝(`.then()`)보다는 가독성을 위해 **`async / await`** 패턴 사용을 지향합니다.
- 단, 병렬 처리를 위한 **`Promise.allSettled`** 사용 시에는 예외입니다.

### 3.4 주석 (Comments)

- 함수 단위로 JSDoc 스타일의 주석을 달아 파라미터와 반환값을 명시합니다.

```jsx
/**
 * Piston API를 호출하여 코드를 실행합니다.
 * @param {string} code - 사용자 입력 코드
 * @param {string} input - 표준 입력(Stdin)
 * @returns {Promise<object>} - 실행 결과 객체
 */
async function executeCode(code, input) { ... }
```

## 4. Git 커밋 메시지 규칙 (Git Commit Message)

[Semantic Commit Messages](https://www.google.com/search?q=https://gist.github.com/joshbuchea/6f2c5fda8906ad04a6bdd7247a646905) 형식을 따릅니다.

- **형식:** `type: subject`
- **유형(Type):**
    - `feat`: 새로운 기능 추가
    - `fix`: 버그 수정
    - `ui`: CSS 등 디자인 변경
    - `docs`: 문서 수정 (README, PRD 등)
    - `refactor`: 코드 리팩토링 (기능 변경 없음)
- **예시:**
    - `feat: Piston API 병렬 호출 로직 구현`
    - `fix: 공유 URL 디코딩 시 한글 깨짐 문제 해결`
    - `ui: 우측 패널 탭 레이아웃 Bootstrap 적용`
