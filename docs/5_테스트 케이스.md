# 테스트 케이스 (Test Cases)

## 1. 기본 기능 테스트 (Functional Testing)

코드 실행 및 채점 로직의 정확성을 검증합니다.

| **ID** | **구분** | **테스트 시나리오** | **입력 데이터 (Input)** | **예상 결과 (Expected Result)** |
| --- | --- | --- | --- | --- |
| **TC-01** | **정답 처리** | 정상 코드와 올바른 예상 출력값 입력 후 실행 | 코드: `print(input())`
Stdin: `Hello`
Expected: `Hello` | 1. 결과 상태가 **'Pass'**로 표시됨.
2. 실행 시간이 표시됨 (예: 0.05s). |
| **TC-02** | **오답 처리** | 정상 코드이나 예상 출력값과 다를 경우 실행 | 코드: `print(10)`
Expected: `20` | 1. 결과 상태가 **'Fail'**로 표시됨.
2. **Diff 영역**에 '내 출력(10)'과 '예상 출력(20)'이 비교되어 표시됨. |
| **TC-03** | **공백 처리** | 출력값에 불필요한 공백/줄바꿈이 포함된 경우 (Trim 검증) | 코드: `print("Hello \n")`
Expected: `Hello` | 1. `trim()` 로직에 의해 공백이 무시되고 **'Pass'**로 판정됨. |
| **TC-04** | **런타임 에러** | 실행 중 치명적 오류 발생 (ZeroDivision 등) | 코드: `print(1/0)` (Python) | 1. 결과 상태가 **'Error'**로 표시됨.
2. `stdout` 대신 **`stderr` 메시지**(ZeroDivisionError...)가 붉은색으로 출력됨. |
| **TC-05** | **컴파일 에러** | 문법적으로 틀린 코드 실행 (Syntax Error) | 코드: `int main {` (C++ 문법 오류) | 1. 결과 상태가 **'Error'**로 표시됨.
2. 컴파일러 에러 메시지가 출력됨. |

## 2. UI/UX 및 상태 관리 테스트 (UI & State Testing)

다중 탭 관리 및 사용자 인터페이스 동작을 검증합니다.

| **ID** | **구분** | **테스트 시나리오** | **절차 및 조건** | **예상 결과 (Expected Result)** |
| --- | --- | --- | --- | --- |
| **TC-06** | **탭 추가** | `[+]` 버튼을 눌러 테스트 케이스 추가 | 최대 권장 개수(6개)까지 추가 시도 | 1. 새로운 탭(Case N)이 생성됨.
2. 새 탭의 입력창(Stdin/Expected)이 비어있는 상태로 초기화됨. |
| **TC-07** | **탭 삭제** | 중간 순서의 탭(예: Case 2) 삭제 | Case 1, 2, 3 중 Case 2 삭제 | 1. Case 2 탭이 사라짐.
2. 기존 Case 3의 내용이 유지되거나 인덱스가 재정렬됨. |
| **TC-08** | **입력 동기화** | 탭 전환 시 입력 데이터 유지 확인 | Case 1에 데이터 입력 -> Case 2 전환 -> 다시 Case 1 복귀 | 1. Case 1에 입력했던 텍스트가 사라지지 않고 그대로 유지됨. |
| **TC-09** | **병렬 실행 UI** | 다수 케이스 실행 시 로딩 상태 확인 | 3개 이상의 케이스를 동시에 실행 | 1. 모든 결과가 도착하기 전까지 버튼이 **'Running...'** 및 비활성 상태가 됨.
2. 실행 완료 후 버튼이 다시 활성화됨. |

## 3. 공유 및 데이터 보존 (Sharing & Persistence)

Zero-DB 아키텍처의 핵심인 URL 해시 기능을 검증합니다.

| **ID** | **구분** | **테스트 시나리오** | **절차 및 조건** | **예상 결과 (Expected Result)** |
| --- | --- | --- | --- | --- |
| **TC-10** | **URL 생성** | 코드 작성 후 공유 버튼 클릭 | 긴 코드(100줄 이상)와 테스트 케이스 2개 작성 후 공유 | 1. URL 뒤에 `/#data=...` 해시가 생성됨.
2. '클립보드에 복사되었습니다' 알림(Alert/Toast) 표시. |
| **TC-11** | **상태 복원** | 공유된 URL로 새 브라우저 창(시크릿 모드)에서 접속 | 브라우저 캐시가 없는 상태에서 접속 | 1. 작성했던 코드, 선택된 언어, 테스트 케이스 2개의 내용이 **완벽하게 복원**됨. |
| **TC-12** | **한글 처리** | 한글 주석이나 한글 문자열이 포함된 코드 공유 | 코드: `print("안녕하세요")` | 1. URL 인코딩/디코딩 과정에서 한글 깨짐 없이 정상 복원됨. |
